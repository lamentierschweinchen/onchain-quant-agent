# MultiversX On-Chain Quant Agent

Weekly intelligence reports for the MultiversX blockchain, generated by a scheduled Claude Code remote agent.

## What It Does

Every Monday at 9:00 AM CET, a remote Claude agent:

1. Queries the MultiversX public API for network economics, whale activity, staking data, token metrics, and DeFi activity
2. Resolves addresses to human-readable names using a curated [known addresses database](data/known-addresses.json) (135+ tagged entities)
3. Computes week-over-week deltas against the [previous snapshot](data/previous.json)
4. Detects anomalies via z-score analysis
5. Generates both a [markdown report](docs/report-template.md) and a structured JSON report (following the [report schema](data/report-schema.json))
6. Commits everything to `reports/` and updates the snapshot

## Report Sections

| Section | What It Covers |
|---------|---------------|
| **TL;DR** | Top 5 findings, one line each |
| **Risk Dashboard** | At-a-glance heatmap of key metrics |
| **Network Health** | EGLD price, staked ratio, APR, supply dynamics, account growth |
| **Whale Intelligence** | Large transactions (>5K EGLD), wallet balance changes, exchange flows, dormant activations |
| **Staking Power Map** | Provider rankings, delegation shifts, concentration metrics (Herfindahl index) |
| **Token & DeFi Activity** | Top ESDTs by holders/volume, xExchange summary, new token issuances |
| **Anomaly Alerts** | Metrics deviating >2 standard deviations from trailing average |
| **Watch List** | Developing patterns tracked across weeks |

## Setup: Scheduled Remote Agent

### Via claude.ai/code/scheduled

1. Go to [claude.ai/code/scheduled](https://claude.ai/code/scheduled)
2. Create a new scheduled agent with:
   - **Name**: `MultiversX Weekly Intel Report`
   - **Schedule**: `0 7 * * 1` (Every Monday 7:00 UTC = 9:00 CET)
   - **Model**: `claude-sonnet-4-6`
   - **Repository**: `https://github.com/lamentierschweinchen/onchain-quant-agent`
   - **Tools**: Bash, Read, Write, Edit, Glob, Grep
3. For the prompt, paste the contents of [`AGENT_PROMPT.md`](AGENT_PROMPT.md)

### Via Claude Code CLI

```bash
claude /schedule create \
  --name "MultiversX Weekly Intel Report" \
  --cron "0 7 * * 1" \
  --repo "https://github.com/lamentierschweinchen/onchain-quant-agent"
```

Then paste the contents of `AGENT_PROMPT.md` as the agent prompt.

## Data Sources

All data comes from the **MultiversX public API** (`api.multiversx.com`) — no authentication required.

| Endpoint | Data |
|----------|------|
| `/economics` | EGLD price, supply, staked ratio, APR |
| `/stats` | Accounts, transactions, epochs, blocks |
| `/accounts?sort=balance` | Top wallets |
| `/transactions?after=...` | Recent activity |
| `/tokens?sort=accounts` | Token holder rankings |
| `/tokens?sort=transactions` | Token activity rankings |
| `/providers?sort=locked` | Staking provider rankings |
| `/identities` | Validator identity metadata |
| `/mex/economics` | xExchange DEX metrics |
| `/mex/pairs` | Trading pair data with volume |
| `/mex/tokens` | DEX-traded tokens with 24h changes |

## Configuration

`config/default.json` controls analysis thresholds:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `analysis.whaleThreshold` | 10,000 EGLD | Minimum balance to classify as whale |
| `analysis.largeTransactionThreshold` | 5,000 EGLD | Minimum tx value for whale intelligence section |
| `analysis.significantMovementUsd` | $100,000 | USD threshold for significant movements |
| `analysis.anomalyZScoreThreshold` | 2.0 | Standard deviations for anomaly flagging |
| `analysis.dormantThresholdDays` | 180 | Days of inactivity before "dormant" classification |
| `analysis.exchangeFlowEnabled` | true | Enable exchange net flow calculation |

## Future: Dashboard

The structured JSON reports (`reports/YYYY-MM-DD.json`) follow a [documented schema](data/report-schema.json) designed for dashboard consumption. The schema covers all report sections with typed fields, making it straightforward to build a visualization layer on top.
